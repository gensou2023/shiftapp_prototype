## 1. デモアプリの概要

### 何ができるアプリか

ShiftOptimizer AI は、**倉庫業務のアルバイトシフトをAIで自動最適化する**デモアプリです。

倉庫管理者は日々、「明日の出荷量が多いから人を増やさないと」「この人は連勤が続いている」「ベテランと新人のバランスを取りたい」といった判断を手作業で行っています。このアプリは、出荷量レベルを選ぶだけで、スタッフのスキル・出勤可能日・コストを考慮した最適なシフトを自動生成します。

### デモの位置づけ

- これは**営業デモ用のプロトタイプ**です
- 表示されている氏名・所属等は**すべて架空のサンプルデータ**です
- 実際のアプリ開発時には、データベース・認証・API連携を追加します
- デモ内の「松竹梅」3プランで、クライアントの予算と要件に合わせた提案が可能です

### データの構成

| データ | 件数 | 説明 |
|--------|------|------|
| スタッフ | 85名 | 4部署（出荷・検品 / 入荷・仕分け / 物流・加工 / 夜間作業） |
| スキルレベル | 1〜5 | 1: 新人、3: 中堅、5: ベテラン |
| 時間帯 | 4種類 | 早番（8:00〜17:00）、遅番（15:00〜22:00）、夜勤（22:00〜翌7:00）、通し（8:00〜22:00） |
| 出荷量レベル | 4段階 | 軽量（スコア20）、通常（40）、繁忙（60）、超繁忙（80） |

## 2. 画面別の使い方ガイド

### 2.1 ダッシュボード

アプリを開くと最初に表示される画面です。本日のシフト状況が一目でわかります。

#### KPIカード（画面上部）

3つのカードが横並びで表示されます。

| カード | 表示内容 | 補足 |
|--------|---------|------|
| 登録スタッフ | 総スタッフ数と稼働状況 | 「82名 稼働中 / 3名 休止」のように、出勤可能日が未登録のスタッフを「休止」として表示します |
| 今週のシフト | 週間の延べ配置人数と充足率 | カレンダーデータから自動計算されます |
| 本日の出荷レベル | 当日の出荷量レベル | 「通常」「繁忙」などが色付きで表示されます |

#### サンプルデータバナー

KPIカードの上に、薄い背景のバナーで「このデモアプリに表示されている氏名・所属等は、すべて架空のサンプルデータです」と表示されます。クライアントに誤解を与えないための表記です。

#### クイックアクションボタン

- **「AIでシフトを生成」** — シフト生成画面に移動します
- **「スタッフ一覧」** — スタッフ一覧画面に移動します

#### 本日のシフトタイムライン

6:00〜翌8:00の時間軸上に、早番・遅番・夜勤の3つのバーが表示されます。

- 各バーの長さは実際の勤務時間帯に対応
- バー内に配置人数が表示されます（例: 「8名」）
- バーの右側に「充足」または「人員不足」のステータスが色付きで表示されます
- **赤い縦線**が現在時刻を示します（「現在」ラベル付き）

これにより、「今この瞬間、倉庫に何人いるのか」が直感的にわかります。

#### 直近の通知（アラートフィード）

以下のような通知が最大5件表示されます。

| 通知タイプ | 例 | アイコン色 |
|-----------|-----|----------|
| 人員不足（危険） | 「本日 2/19(水) 遅番が人員不足です」 | 赤 |
| 注意 | 「明日 2/20(木) 夜勤が人員不足の見込み」 | 黄 |
| 労務警告 | 「マリア ロサ、赤木 茂 他1名 — 週6日以上の出勤登録」 | 黄 |
| 確定通知 | 「2/19(水) 早番のシフトが確定済みです」 | 緑 |
| お知らせ | 「スタッフ登録数が85名になりました」 | 青 |

人員不足の通知には **「追加人員を募集」ボタン** が表示されます（現在は開発中機能のプレースホルダー）。

#### 3スロットサマリーカード

画面下部に、早番・遅番・夜勤の3枚のカードが並びます。

各カードに含まれる情報：
- **スロット名と時間帯**（例: 早番 8:00〜17:00）
- **充足/人員不足バッジ**
- **配置スタッフ一覧**（名前チップで表示。7名以上は「+2名」と省略）
- **スコアバー**（スコア達成率をプログレスバーで表示。充足=緑、不足=赤）

**注意:** 各スロットのスタッフは重複しないよう制御されています。早番に配置されたスタッフは遅番・夜勤の候補から除外されるため、実際の運用に近い表示になります。

### 2.2 スタッフ一覧

85名のスタッフ情報をカード形式で一覧表示する画面です。

#### ヘッダー操作

- **「スタッフ追加」ボタン** — 新規スタッフ登録モーダルが開きます
- **「CSV取込」ボタン** — CSVファイルから出勤可能日・時間帯を一括更新できます

#### 検索バー

画面上部の検索欄に文字を入力すると、**スタッフ名・部署名・備考**でリアルタイムにフィルタリングされます。

例: 「フォークリフト」と入力 → フォークリフト資格保持者のみ表示

#### 部署フィルタ

「全て」「出荷・検品」「入荷・仕分け」「物流・加工」「夜間作業」のタブで部署別に絞り込めます。各タブの横に人数が表示されます。

#### スタッフカード

各カードには以下の情報が表示されます。

| 要素 | 内容 |
|------|------|
| アバター | 名前の頭文字（色はランダム） |
| 名前 | フルネーム |
| 部署バッジ | 色分けされたバッジ（出荷=青、入荷=緑、物流=紫、夜間=紺） |
| 勤続年数バッジ | 「勤続3年」のように表示 |
| スキルレベル | 星5つ（★★★☆☆ = Lv.3） |
| 出勤可能曜日 | 月〜日のバッジ（出勤可能な日のみ表示） |
| 出勤可能時間帯 | 早番・遅番・夜勤・通しのバッジ |
| 時給 | 例: ¥1,200/時 |
| 貢献度 | 0.5〜2.0（シフト生成の重み付けに使用） |
| 備考 | 「班長・フォークリフト資格」「扶養内・シフト制」など |

#### カードクリック → 編集モーダル

カードをクリックすると「スタッフ編集」モーダルが開きます。

**編集可能な項目:** 名前、部署、スキルレベル、時給、貢献度、出勤可能曜日（チェックボックス）、出勤可能時間帯（チェックボックス）、備考

**閲覧専用の詳細情報:**

- **入社日と勤続年数** — スタッフIDから決定論的に生成（同じIDなら常に同じ値）
- **スキル推移** — 入社時Lv.1 → いつLv.2に上がったか、のタイムライン表示
- **部署異動履歴** — 異動経験がある場合は前部署 → 現部署の履歴

#### CSV取込機能

「CSV取込」ボタンをクリックすると専用モーダルが開きます。

**CSVフォーマット:**
```
名前,出勤可能曜日,出勤可能時間帯
佐藤 浩司,月火水木金,早番遅番
加藤 隆太,土日,早番
```

**使い方:**
1. CSVファイルを選択
2. プレビュー画面で「一致」「不一致」を確認（名前がスタッフ一覧と一致するかチェック）
3. 「適用」ボタンで一括更新
4. 不一致のスタッフはスキップされます

### 2.3 AIシフト生成

出荷量に応じた最適なシフトをAIが自動生成するメイン機能です。

#### 入力条件パネル（左側）

| 入力項目 | 操作 | デフォルト |
|---------|------|----------|
| 日付 | カレンダーから選択 | 本日 |
| 時間帯 | 早番/遅番/夜勤/通し の4択 | 早番 |
| 出荷量レベル | 軽量/通常/繁忙/超繁忙 の4択 | 通常 |

入力条件の下に「必要スコア」と「出勤可能スタッフ数」がリアルタイムで表示されます。

**「AIでシフト生成」ボタン**をクリックすると、分析アニメーション → 結果表示の流れになります。

#### 結果表示: 推奨案（効率重視）

最も効率的なシフト案が最初に表示されます。

**表示される情報:**
- **日付・時間帯・出荷量レベル** — 入力条件の確認
- **スコア達成率** — プログレスバー（緑=充足、赤=人員不足）
- **配置スタッフ数** — 例: 「8名配置」
- **連勤・労務チェック警告** — 該当者がいれば黄色の警告ボックスで表示
  - 例: 「マリア ロサ: 6連勤目の可能性（要確認）」
- **スタッフチップ** — 配置される各スタッフの名前・スキルレベルが順番にアニメーション表示
- **Ganttタイムライン** — 各スタッフの実際の勤務時間帯をバーチャートで表示
- **人件費概算** — 例: 「人件費概算（4時間）: ¥36,800」

**操作可能な機能:**

1. **欠勤シミュレーション** — スタッフチップをクリックすると、そのスタッフが欠勤した場合の影響パネルが表示されます
   - スコア影響: 「45.2 → 35.2 (-10.0)」
   - 代替候補3名が自動提案（スキル×貢献度の降順）
   - 各候補の「スコア回復: +X.X」を表示

2. **シフト確定** — 「このシフトを確定する」ボタンをクリック
   - ボタンが「確定済み ✓」に変化
   - 画面上部にトースト通知:「2026/2/19（水）早番のシフトを確定しました」
   - カレンダー画面にも反映されます

3. **Excel出力** — 「Excel出力」ボタン（現在は開発中機能プレースホルダー）

#### 代替案の表示

「代替案を表示（コスト最小・バランス重視）」をクリックすると、2つの追加案が展開されます。

| 案 | アルゴリズム | 特徴 |
|----|-----------|------|
| A: 効率重視 | スキル×貢献度の高い順に配置 | 最少人数で目標スコア達成 |
| B: コスト最小 | 時給の低い順に配置 | 人件費を最小化 |
| C: バランス重視 | ベテラン1名 + 高スキル/低スキルを交互に配置 | 教育効果を考慮 |

#### 1日表示 / 週間表示の切替

結果エリアの上部に表示切替ボタンがあります。

**週間表示**を選択すると、月曜〜土曜の6日分のシフトがグリッドで一括表示されます。

各日のセルに表示される情報:
- 曜日と日付
- 出荷量レベル（軽量/通常/繁忙）
- 充足/人員不足バッジ
- 配置人数とスコア

セルをクリックすると、その日の詳細シフト生成に切り替わります。

### 2.4 シフトカレンダー

月間のシフト状況を一覧表示する画面です。

#### カレンダーグリッド（PC表示）

月間カレンダーが7列のグリッドで表示されます。各日付のセルには以下が表示されます。

| バッジ | 意味 | 色 |
|--------|------|-----|
| 充足 | 必要スコアを満たしている | 緑 |
| 不足気味 | 仮のシフト（あと少し人が足りない） | 黄 |
| 人員不足 | 大幅に人が足りない | 赤 |
| 休み | 日曜（休業日） | グレー |

**ポイント:** カレンダーのステータスは曜日に基づいて現実的に分布しています。月曜・金曜は繁忙になりやすく、水曜は軽量になりやすい、といったパターンが反映されています。

#### カレンダーリスト（モバイル表示）

スマホでは、日付を縦にリスト表示します。同じ情報がコンパクトに確認できます。

#### 日付詳細パネル

日付をクリックすると、画面下部に詳細パネルがスライドインします。

表示内容:
- 出荷レベル
- ステータス（確定/仮/人員不足）
- 配置人数
- スコア
- 配置スタッフ一覧（名前とスキルレベル）

**操作:**
- **「シフト生成」ボタン** — クリックすると、シフト生成画面にその日付が自動セットされた状態で移動します。人員不足の日をカレンダーで見つけて、すぐにシフト生成を実行できます
- 人員不足の日には **「追加人員を募集」ボタン** が表示されます（開発中機能）

### 2.5 設定

画面右上の歯車アイコンをクリックすると設定モーダルが開きます。3つのタブがあります。

#### 表示設定タブ

| 設定項目 | 説明 |
|---------|------|

| 設定項目 | 説明 |
|---------|------|
| シフト変更通知 | シフトが変更された場合に通知します |
| 労務警告通知 | 連勤超過やインターバル不足の警告 |
| 欠勤通知 | スタッフが欠勤した場合に通知します |

### シフト生成アルゴリズムの詳細

#### A. 効率重視（推奨案）

```
1. 出勤可能スタッフを「スキルレベル × 貢献度」の降順でソート
2. 上位から順に追加し、合計スコアが必要スコアに達したら終了
→ 最少人数で目標スコアを達成
```

**メリット:** 最小限の人数で高品質なシフトを実現
**デメリット:** ベテランに偏りやすく、新人の育成機会が減る

#### B. コスト最小

```
1. 出勤可能スタッフを時給の昇順でソート
2. 低コスト順に追加し、合計スコアが必要スコアに達したら終了
→ 人件費を最小化
```

**メリット:** コストを最小限に抑えられる
**デメリット:** スキルの低いスタッフが多くなり、品質低下の可能性

#### C. バランス重視

```
1. スキル3以上（高スキル）と2以下（低スキル）に分割
2. まずベテラン1名を配置
3. 高スキル・低スキルを交互に配置し、スコアが目標に達するまで繰り返す
→ ベテランと新人のバランス配置
```

**メリット:** ベテランが新人をサポートする体制、教育効果が期待できる
**デメリット:** 人数が多くなりがちで、コストは効率重視より高い

## 4. 松竹梅プランと本番開発への道筋

このデモは「竹プランの簡易版」に相当します。クライアントの予算と要件に応じて、3段階のプランを提案します。

### プラン比較表

| 機能 | 梅（エージェント型） | 竹（Webアプリ基本版） | 松（AI学習型フル版） |
|------|:---:|:---:|:---:|

### 梅プラン — エージェント型

- **構成:** Google Apps Script / Power Automate + スプレッドシート + チャットbot
- **向いているケース:** まず小さく始めたい、IT予算が限られている
- **できること:** 基本的なシフト自動生成、スプレッドシートでスタッフ管理

### 竹プラン — Webアプリ基本版（今回のデモ相当）

- **構成:** フロントエンド + サーバーレスバックエンド + DB
- **向いているケース:** しっかりしたUIで管理したい、スマホからアクセスしたい
- **できること:** デモで見せた全機能を本番品質で実装

### 松プラン — AI学習型フル機能版

- **構成:** Next.js + Supabase + AI推論エンジン
- **向いているケース:** シフト管理を本格的にDXしたい、データ資産を活かしたい
- **できること:** 竹の全機能 + 過去データ学習 + 予測 + 高度な最適化 + レポート

### 推奨ステップ

```
Step 1: デモを見てイメージを共有
Step 2: 竹プランでまず使えるものを作る（1〜2ヶ月）
Step 3: 運用データが溜まったら松プランへ段階的にアップグレード
```

## 5. AI活用の方針 — 本番アプリでどうAIを使うか

### 「AIを入れる」とは具体的に何をするのか

シフト管理における AI の活用は、大きく **3つのレイヤー** に分けられます。

#### レイヤー1: 予測ML（機械学習）

過去のデータからパターンを学習し、未来を予測するAI。

| 予測対象 | 使うデータ | アウトプット |
|---------|-----------|------------|
| **出荷量予測** | 過去の出荷量 + 曜日 + 季節 + イベント | 「来週月曜は繁忙レベルになる可能性80%」 |
| **欠勤予測** | 過去の欠勤パターン + 天候 + 曜日 | 「佐藤さんは月曜の出勤率が70%」 |
| **離職リスク** | 勤務パターン + 出勤率の変化 | 「加藤さんの出勤頻度が低下傾向」 |

**技術:** 時系列予測（Prophet, LightGBM）、分類モデル
**必要データ:** 3ヶ月〜6ヶ月以上の蓄積データ

#### レイヤー2: 最適化エンジン

制約条件を満たしながら、最適な組み合わせを見つけるAI。

| 最適化対象 | 制約条件 | アウトプット |
|-----------|---------|------------|
| **シフト配置** | 連勤制限、インターバル確保、公平性 | 全ての制約を満たす最適シフト |
| **コスト最適化** | 予算上限、スキル要件、労務規定 | コストと品質のバランス最適解 |

**技術:** 数理最適化（PuLP, Google OR-Tools）、制約充足問題
**特徴:** データの蓄積は不要。ルールを定義すれば即座に動作

#### レイヤー3: 生成AI（LLM）

自然言語で対話し、判断を支援するAI。

| 活用場面 | 例 |
|---------|-----|
| **自然言語でのシフト調整** | 「来週の月曜、佐藤さんを早番から遅番に変えて」 |
| **レポート生成** | 「今月のシフト充足率と課題を要約して」 |
| **判断支援** | 「この出荷量で8名は適切？根拠を教えて」 |
| **異常検知の説明** | 「なぜこのスタッフに警告が出ているのか？」 |

**技術:** Claude API / OpenAI API + RAG（社内データ参照）
**特徴:** ユーザーとの対話を通じて高度な判断を支援

### 松プランでの具体的なAI活用イメージ

```
■ ユーザーの操作フロー（松プラン）

1. アプリを開く
   → AIが「来週の出荷量予測」を自動表示
   → 「月曜は繁忙（確信度85%）、水曜は軽量（確信度70%）」

2. シフト生成ボタンをクリック
   → 最適化エンジンが制約条件を考慮してシフト案を生成
   → 3案比較（効率/コスト/バランス）は今のデモと同じ
   → 加えて「連勤リスク」「公平性スコア」も表示

3. 気になることをチャットで質問
   → 「佐藤さんの今月の稼働時間は多い？」
   → AIが過去データを参照して回答
   → 「佐藤さんの今月の稼働は82時間で、平均の1.2倍です。
      来週は休みを推奨します」
```

### RAG + LLM のアプローチ（別案）

専門的なMLモデルを個別に開発する代わりに、**RAG（検索拡張生成）+ LLM** で代替するアプローチもあります。

```
■ RAG + LLM の構成

[過去のシフトデータ] → ベクトルDB化（Embedding）
[出荷実績データ]     → ベクトルDB化
[スタッフ勤怠データ]  → ベクトルDB化

ユーザーの質問・シフト生成リクエスト
  ↓
LLM（Claude API）が関連データを検索（RAG）
  ↓
データに基づいた回答・シフト提案を生成
```

**RAG + LLM のメリット:**
- 個別のMLモデル開発が不要（開発コスト削減）
- 自然言語での対話が最初から可能
- データが少なくても動作する（10〜30件程度でも参照可能）

**RAG + LLM の限界:**
- 数値精度は専門MLに劣る（「確信度85%」のような精密な予測は難しい）
- 最適化問題の厳密な求解には不向き
- API呼び出しコストが継続的に発生

**推奨:** 小〜中規模の運用ではRAG + LLMで十分実用的。規模が大きくなった場合に、予測MLや最適化エンジンを段階的に追加する。

### 段階的なAI導入ロードマップ

```
Phase 1（竹プラン）
  └─ ルールベースの自動生成（今のデモの本番版）
  └─ 欠勤時の代替提案
  └─ 連勤・労務チェック

Phase 2（竹→松の中間）
  └─ RAG + LLM でチャット型支援を追加
  └─ 「先月のシフト傾向を教えて」→ データに基づく回答
  └─ 自然言語でのシフト調整指示

Phase 3（松プラン）
  └─ 出荷量の予測ML導入（過去データ蓄積後）
  └─ 最適化エンジンによる制約考慮
  └─ 月次レポート自動生成
  └─ 離職リスク・欠勤パターン分析
```

## 6. クライアントへのデモ方針

### デモのストーリーライン

クライアント（倉庫管理者）に対して、以下の順序でデモを進めることを推奨します。

#### 1. 課題の共感（1分）

> 「毎月のシフト表、手作業で大変ですよね。85名のスタッフのスキル・出勤可能日を頭の中で組み合わせて...」

#### 2. ダッシュボードで全体像を見せる（2分）

- KPIカードで「登録スタッフ85名」を見せ、規模感を伝える
- タイムラインで「今日のシフトがぱっと見でわかる」ことをアピール
- アラートで「人員不足に気づける」ことを見せる

#### 3. スタッフ一覧で「データの細かさ」を見せる（2分）

- 部署フィルタで切り替え、「こんなに細かく管理できる」
- カード1枚をクリックして、スキル推移・部署異動を見せる
- 「このデータを元に、AIが最適なシフトを考えます」と繋げる

#### 4. シフト生成がメインの見せ場（5分）

- 出荷量レベルを「繁忙」に設定
- **「AIでシフト生成」をクリック** → アニメーション → 結果表示
- スコアバーで「目標達成」を確認
- Ganttタイムラインで「誰が何時から何時まで」を確認
- **欠勤シミュレーション**: スタッフチップをクリック → 「もしこの人が休んだら？」→ 代替候補3名を見せる
- **確定ボタン**: 「よければ確定」→ トースト表示
- 代替案を展開: 「コスト重視ならこう、バランス重視ならこう」

#### 5. カレンダーで全体を俯瞰（1分）

- 月間カレンダーで「人員不足の日が一目でわかる」
- 赤い日をクリック → 「この日はシフト生成が必要ですね」→ シフト生成へ飛ぶ

#### 6. プラン提示（2分）

> 「今見ていただいたのは竹プランの簡易版です。まずここから始めて、データが溜まったら松プランでAIの予測機能を追加できます」

### デモ時のポイント

| ポイント | 説明 |
|---------|------|
| **操作はシンプルに** | 出荷量を選んでボタンを押すだけ、と強調 |
| **数字で語る** | 「スコア45.2で目標40を達成」のように具体的に |
| **「もしも」を見せる** | 欠勤シミュレーションで「想定外にも対応できる」 |
| **拡張性を匂わせる** | 「これは基本版で、AIの予測を入れるとこうなります」 |
| **現場感** | 4部署・早番/遅番/夜勤の構成がリアルであることを伝える |

### 営業担当向けの補足

- **「AIって具体的に何？」と聞かれたら:** 「今のデモはルールベースの自動計算ですが、本番ではお客様の過去3ヶ月分のシフトデータを学習させて、出荷量の予測や欠勤パターンの分析もできるようになります」
- **「いくらかかるの？」と聞かれたら:** 松竹梅の表を見せて、「まずは竹プランで始めて、効果が見えたら段階的にアップグレードするのが安全です」
- **「うちのデータでも動く？」と聞かれたら:** 「CSV取込機能がありますので、今お使いのExcelのシフト表からデータを取り込めます」
