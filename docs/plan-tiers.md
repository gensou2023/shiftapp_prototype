# シフト最適化 — 松竹梅プラン

## 概要

倉庫業務のアルバイトシフト最適化を、3段階のプランで提案する。
クライアントの予算・技術環境に応じて選択可能。

---

## 梅プラン — エージェント型（ローコスト）

### 概要
既存の Google Workspace / Microsoft 365 環境内で完結するエージェント構築。
新規インフラ不要で、最小コストで AI シフト最適化を実現。

### 構成
- **Google Apps Script** or **Power Automate** でシフトロジックを自動化
- **スプレッドシート**（Google Sheets / Excel）をデータベース兼UIとして活用
- **チャットbot**（Google Chat / Teams）で出荷量入力 → シフト案返却

### できること
- スタッフ一覧管理（スプレッドシート）
- 出荷量入力 → 基本的なシフト自動生成
- 結果をスプレッドシート or チャットで返却

### できないこと
- リッチなUI（カレンダー表示等）
- 複雑な最適化アルゴリズム
- 過去データからの学習

### 向いているケース
- まず小さく始めたい
- IT予算が限られている
- 既にGoogle / Microsoft環境を使っている

---

## 竹プラン — Webアプリ基本版

### 概要
専用のWebアプリを構築。スタッフ管理・シフト自動生成・カレンダー表示を提供。
**今回のデモはこのプランの簡易版。**

### 構成
- **フロントエンド:** HTML/CSS/JS（将来的にNext.js等へ移行可能）
- **バックエンド:** サーバーレス（Cloudflare Workers / Vercel Functions）
- **データベース:** Supabase or Cloudflare D1
- **デプロイ:** Cloudflare Pages

### できること
- スタッフ一覧管理（登録・編集・スキルレベル設定）
- 出荷量スコアに基づくシフト自動生成
- シフトカレンダー表示（月間・週間）
- スマホ対応（倉庫現場で閲覧可能）
- Excel出力（シフト表ダウンロード）

### できないこと
- 過去データからの自動学習
- 高度な制約条件（連勤制限・公平性最適化等）

### 向いているケース
- しっかりしたUIで管理したい
- スマホからアクセスしたい
- 将来的な機能拡張を見据えたい

---

## 松プラン — AI学習型フル機能版

### 概要
竹プランの全機能に加え、過去シフトデータからの学習・高度な最適化を実現。
倉庫業務の実運用に耐えるフルスペック版。

### 構成
- **フロントエンド:** Next.js（React）
- **バックエンド:** Next.js API Routes + AI推論
- **データベース:** Supabase PostgreSQL
- **AI:** OpenAI API or Claude API（最適化エンジン）
- **デプロイ:** Vercel

### できること（竹プランに加えて）
- **過去シフトデータの学習** — 3ヶ月分のシフト表をインポート、パターンを学習
- **高度な制約条件** — 連勤制限、スキルバランス、公平性の最適化
- **予測** — 出荷量の傾向予測 → 事前シフト案の自動提案
- **複数案比較** — コスト最小案 / バランス重視案 / 品質重視案の3案を同時提示
- **レポート** — 月間シフト分析レポート（コスト、稼働率、偏り）
- **チーム管理** — 管理者・リーダー・スタッフの権限管理

### 向いているケース
- シフト管理を本格的にDXしたい
- 過去のデータ資産を活かしたい
- 複数拠点への展開を検討している

## 推奨ステップ

1. **まず梅プランで業務フローを検証**（1-2週間）
2. **竹プランでUI/UXを整備**（効果が見えたら）
3. **松プランで本格運用**（投資判断後）

または、今回のデモ（竹の簡易版）を見て頂き、ご予算と優先度に応じて決定。
