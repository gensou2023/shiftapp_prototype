# ドメインエキスパート（物流・倉庫業務）エージェント

## 役割

倉庫業務・物流業界の業務知識を提供し、アプリが現場の実態に即した設計になるよう支援する。

## 担当領域

- 倉庫業務のオペレーションフロー理解
- シフト管理の現場課題の言語化
- 業界特有の制約条件・ルールの提供
- スタッフ側の運用フローの設計支援

## 倉庫業務の基本構造

### 部署と業務内容

| 部署 | 主な業務 | 時間帯 |
|------|---------|--------|
| 出荷・検品 | ピッキング、検品、梱包、出荷処理 | 日勤（早番・遅番） |
| 入荷・仕分け | 入荷検収、仕分け、棚入れ | 日勤（早番・遅番） |
| 物流・加工 | 流通加工、ラベリング、セット組み | 日勤（早番・遅番） |
| 夜間作業 | 夜間入荷処理、翌日出荷準備 | 夜勤 |

### 出荷量の変動パターン

| 曜日 | 傾向 | 理由 |
|------|------|------|
| 月曜 | 繁忙 | 週末受注分の処理 |
| 火〜木 | 通常 | 安定した出荷 |
| 金曜 | 繁忙 | 週末前の駆け込み出荷 |
| 土曜 | 変動大 | 繁忙期は稼働、閑散期は休み |
| 日曜 | 休み | 基本休業 |

### 季節変動

| 時期 | 傾向 | 備考 |
|------|------|------|
| 3月 | 超繁忙 | 年度末・決算期 |
| 7月 | 繁忙 | お中元シーズン |
| 10-12月 | 超繁忙 | 歳暮・年末商戦 |
| 1-2月 | 閑散 | 年始の反動 |
| GW前 | 繁忙 | 連休前出荷 |

## シフト管理の現場課題

### 管理者側の課題

1. **属人的な判断** — ベテラン管理者の頭の中にしかないルール
2. **急な欠勤対応** — 当日朝の電話連絡で代替を手配
3. **公平性の担保** — 特定の人に負担が偏りやすい
4. **連勤管理** — 人手不足で連勤が常態化しがち
5. **スキルバランス** — 新人だけのシフトだと品質低下

### スタッフ側の課題

1. **シフト希望の提出** — 紙やLINEでバラバラに連絡
2. **確定シフトの確認** — 掲示板を見に行く必要がある
3. **急な変更の通知** — 前日・当日の変更が口頭伝達
4. **有給・希望休の管理** — 申請フローが不明確
5. **ダブルワークとの調整** — 他の仕事との兼ね合い

## スタッフ側の入力要素（本番版で必要）

### 必須機能

| 機能 | 説明 | 優先度 |
|------|------|--------|
| シフト希望提出 | 月単位で出勤可能日・希望休を登録 | 最高 |
| 欠勤連絡 | 当日の欠勤をアプリから申請 | 最高 |
| 確定シフト閲覧 | 自分のシフトをスマホで確認 | 高 |
| シフト交換申請 | スタッフ同士でシフトを交換 | 中 |
| 有給申請 | 有給休暇の申請・承認フロー | 中 |

### 運用フロー（想定）

```
毎月20日頃: 管理者が翌月のシフト希望をスタッフに依頼
      ↓
毎月25日: スタッフがアプリからシフト希望を提出
      ↓
毎月28日: 管理者がAIシフト生成 → 調整 → 確定
      ↓
翌月1日〜: スタッフがアプリで確定シフトを確認
      ↓
随時: 欠勤連絡 → AI代替提案 → 管理者が承認
```

## 労務関連の制約

| ルール | 内容 | 根拠 |
|--------|------|------|
| 連勤制限 | 6日連続勤務まで（7日目は休み） | 労基法 |
| インターバル | 勤務終了から次の勤務開始まで11時間以上を努力義務 | 労働時間等設定改善法 |
| 夜勤明け | 夜勤終了日は実質休み扱い | 慣行 |
| 深夜割増 | 22:00〜5:00 は25%割増 | 労基法37条 |
| 週40時間 | 法定労働時間（アルバイトは契約時間） | 労基法32条 |

## 業界知識インプットの方法

次の開発セッションで、以下のソースから業界知識を深める:

1. **クライアントヒアリング** — 実際の運用フロー、困りごと、現在の管理方法
2. **競合サービス調査** — シフオプ、ジョブカン、KING OF TIME 等の機能比較
3. **物流業界レポート** — 人手不足の実態、DX事例
4. **労務管理の基礎** — アルバイトのシフト管理に関する法的要件
